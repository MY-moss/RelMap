# 联系人管理

RelMap 提供全面的联系人管理能力，支持丰富的字段和灵活的扩展。

## 联系人字段

| 字段 | 说明 |
|------|------|
| 姓名 | 必填，联系人的姓名 |
| 昵称 | 可选的昵称或别名 |
| 生日 | 生日日期，设置后自动创建年度生日提醒 |
| 性别 | 0-未知 / 1-男 / 2-女 |
| 公司 | 所属公司 |
| 职位 | 职位/头衔 |
| 部门 | 所在部门 |
| 备注 | 自定义备注信息 |
| 收藏 | 标记为收藏联系人 |
| 归档 | 归档联系人（隐藏在主列表） |

## 社交账号

每个联系人可以绑定多个社交账号，支持的字段包括：

- 平台名称（如微信、微博、LinkedIn 等）
- 账号 ID
- 账号显示名称
- 是否为主要账号

## 分组

将联系人分配到不同的分组以便管理：

- 一个联系人可以属于多个分组
- 分组支持颜色标识
- 分组内有角色区分（owner / admin / member）

## 标签

标签提供更灵活的分类方式：

- 标签支持颜色标识
- 标签支持层级关系（父标签）
- 标签可应用于联系人、事件、日记等不同实体
- 每个实体可以打多个标签

## 生命周期阶段

联系人可设置生命周期阶段，帮助了解关系热度：

| 阶段 | 说明 |
|------|------|
| `new` | 新认识 |
| `active` | 活跃联系 |
| `maintain` | 维持阶段 |
| `dormant` | 休眠阶段 |
| `lost` | 已失联 |
| `archived` | 已归档 |

## 搜索

支持全局搜索，搜索范围包括联系人姓名、昵称、公司、职位、备注等字段。

支持语义搜索，通过 Ollama 集成自然语言查询能力。
