# 安全设置

RelMap 高度重视用户数据的隐私和安全。所有数据默认存储在本地，不经过任何外部服务器。

## PIN 锁

为应用设置 PIN 码锁定，防止未授权访问：

- **设置 PIN** — 输入 4 位或以上的 PIN 码
- **验证 PIN** — 每次启动应用时验证
- **移除 PIN** — 清除已设置的 PIN 码

PIN 码使用 `PBKDF2`（100,000 次迭代，SHA-512）进行哈希存储，确保即使配置文件泄露也无法还原 PIN。

## BYOK（自带密钥）

备份文件支持密码加密：

- 使用用户提供的密码加密备份文件
- 加密备份使用 `.enc` 扩展名
- 恢复时需要提供相同的密码
- 未加密备份使用 `.db` 扩展名

## 数据隐私

- **本地优先** — 所有数据存储在用户设备本地，无需注册账号
- **无云端同步** — 数据不经过任何外部服务器
- **日志安全** — 错误日志自动脱敏，移除文件路径、姓名、ID 等敏感信息
- **路径安全** — 文件操作使用路径安全检查，防止路径遍历攻击

## 配置存储

应用配置存储在 Electron 的 `userData` 目录下的 `app-config.json` 文件中：

- PIN 哈希和盐值
- AI 服务提供商配置（API Key 等）
- 用户自定义设置

## 健康检查

应用提供健康检查功能，监控：

- 数据库连接状态和延迟
- 内存使用情况
- 应用运行时间
- 数据库完整性校验
